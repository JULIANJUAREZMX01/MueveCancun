## 2025-03-02 - O(1) Route Match Optimization **Learning:** Inefficient string comparison algorithms (Jaro Winkler) applied to nested arrays (every stop of every route) resulted in severe performance bottlenecks in Rust WASM. **Action:** Optimized `match_stop` function by leveraging the existing O(1) `HashMap` (`route.stop_name_to_index`) to resolve exact matches instantly, avoiding costly O(N) fuzzy scanning. Speed improvement of 36,480x observed for valid string queries.

Removed duplicate `src/lib/CoordinatesStore.ts`. Enforced `RouteCalculatorModule` interface in WasmLoader.ts (ADR 003). All WASM artifacts now excluded via `.gitignore`.
## 2026-03-28 - [Merge Conflict Resolution — Data Timestamps] **Learning:** Auto-generated JSON files (routes-index.json, master_routes.optimized.json) that contain timestamps will always conflict when two branches regenerate them independently. **Action:** When resolving timestamp-only conflicts, always prefer the branch with the more recent `generated`/`last_merged` timestamp. Document which branch "owns" the regeneration step to avoid repeated conflicts.
