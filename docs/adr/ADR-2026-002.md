# ADR – Decisión de Arquitectura

**Título:** Uso de Astro Islands + Rust/WASM + Web Components/Lit en MueveCancún
**ID:** ADR-2026-002
**Autor:** Julian
**Fecha:** 13/03/2026
**Versión:** 1.0
**Estado:** Aprobado
**Normatividad aplicable:** ISO 9001, ADR estándar, BPMN

## 1. 🎯 Contexto
El proyecto MueveCancún – ¿Qué Ruta Me Lleva? busca ofrecer rutas de transporte público reales en Cancún, con soporte offline-first y cálculo rápido de rutas.
El stack actual se basa en Rust/WASM para lógica intensiva, TypeScript para tipado, Astro Islands para orquestación, y CSS purista/PostCSS/Houdini para estilos.
Se requiere documentar la decisión de evitar frameworks comunes (React, Tailwind) y adoptar un enfoque más ligero y controlado.

## 2. 🧩 Decisión
Adoptar Astro Islands como capa de presentación, con Rust/WASM para cálculo de rutas y lógica intensiva, y Web Components/Lit para encapsular la UI.
Eliminar dependencias de React y Bootstrap del proyecto.

**Estado actual (transición):** Tailwind CSS sigue activo en producción durante la migración progresiva.
**Estado objetivo:** Tailwind CSS será reemplazado incrementalmente por CSS purista + PostCSS/Houdini, componente por componente, con documentación explícita por cada paso de la migración.

## 3. 📋 Justificación
- **Rendimiento:** WASM optimizado con wasm-opt garantiza tiempos de cálculo < 100ms.
- **Ligereza:** Astro Islands evita runtimes pesados y entrega SSR rápido.
- **Compatibilidad:** Web Components interoperan con cualquier stack y se integran bien con Astro.
- **Control:** Filosofía purista (diógenes.dev.style) se mantiene, con CSS modular y extensiones Houdini.
- **Accesibilidad:** Roles ARIA y WCAG implementados manualmente o con soporte ligero de Lit.

## 4. 📊 Alternativas consideradas
- **React + Tailwind:** descartado por peso y dependencia externa.
- **Svelte:** atractivo por compilación a JS plano, pero innecesario dado que Astro Islands + Web Components cumplen el mismo objetivo.
- **Bootstrap:** descartado por sobrecarga visual y estilos obsoletos.

## 5. ⚠️ Impacto
- **Migración:** eliminación de dependencias React/Tailwind.
- **Capacitación:** equipo debe familiarizarse con Web Components/Lit.
- **Documentación:** actualización de manuales y guías de estilo.
- **Riesgos:** curva de aprendizaje inicial, menor comunidad que React.
- **Mitigación:** uso de Lit y documentación interna clara.

## 6. 📈 Indicadores de éxito (KPIs)
- **Tiempo de cálculo de ruta:** < 100ms en móviles.
- **Tamaño del binario WASM:** ≤ 500KB optimizado.
- **Disponibilidad offline:** ≥ 95% de funcionalidades sin conexión.
- **Lighthouse Performance:** ≥ 90.

## 7. 📜 Historial de cambios
- **v1.0 (13/03/2026):** Decisión inicial de migración a Astro Islands + Rust/WASM + Web Components/Lit.
- **v1.1 (pendiente):** Inclusión de métricas adicionales y casos de prueba.

## 8. ✅ Estado futuro (TO-BE)
- UI basada en Web Components/Lit.
- Estilos gestionados con CSS purista + PostCSS/Houdini.
- Lógica intensiva en Rust/WASM, orquestada en Astro Islands.
- Documentación estandarizada en ADRs y BPMN.
