## 1. 🎯 Contexto

Durante el desarrollo iterativo del proyecto MueveCancún, los agentes de IA y las sesiones de desarrollo acumularon:
- Scripts de parche temporales (`patch_rutas.cjs`, `patch_astro_config.cjs`) comprometidos en el repositorio.
- Documentación contradictoria: docs afirmaban que Tailwind/Bootstrap habían sido eliminados cuando aún siguen activos.
- Mutaciones directas de `globalThis.window` y `globalThis.navigator` en tests que no se restauraban, causando posibles fallos de aislamiento entre suites.
- El build script `scripts/build-wasm.mjs` incluía `spatial-index` en el array `modules` sin que existieran los artefactos WASM correspondientes, rompiendo `pnpm build` en entornos sin wasm-pack.

Se necesitaba una sesión de limpieza y hardening antes del merge de este PR.

---

## 2. 🧩 Decisión

### 2.1 Tests — Aislamiento de Globales

**Decisión**: En `src/tests/telemetry.test.ts`, reemplazar mutaciones directas de `globalThis.window` y `globalThis.navigator` por el patrón `vi.stubGlobal(...)` + `vi.unstubAllGlobals()` en `afterEach`.

**Justificación**: Vitest corre tests en Node.js sin DOM real. Mutaciones directas de `globalThis` persisten entre tests y pueden causar fallos no deterministas dependiendo del orden de ejecución. `vi.stubGlobal` garantiza restauración automática.

**Patrón de referencia**: `src/tests/RouteDrawer.test.ts` (líneas 38-55) — ya usaba este patrón correctamente.

### 2.2 Documentación — Alineación con la Realidad del Código

**Decisión**: Corregir toda documentación que afirmaba que Tailwind había sido eliminado.

**Cambios**:
- `README.md`: "Tailwind" movido a "Dependencias en proceso de eliminación (migración)".
- `docs/ARCHITECTURE.md`: Sección de styling actualizada a "Currently Tailwind CSS + custom CSS; target: CSS purista + PostCSS/Houdini".
- `docs/adr/ADR-2026-002.md`: Aclarado que solo React/Bootstrap han sido eliminados; Tailwind está en transición progresiva.

**Justificación**: La documentación contradictoria con el código activo confunde a futuros contribuidores y agentes de IA. La documentación debe reflejar el estado actual, no el estado objetivo.

### 2.3 Scripts de Parche — Eliminación

**Decisión**: Eliminar `patch_rutas.cjs` y `patch_astro_config.cjs` del repositorio.

**Justificación**: Son scripts one-off generados durante sesiones de desarrollo. No están referenciados en ningún `npm script`, no tienen documentación de uso, y su presencia contradice el objetivo del PR de limpiar artefactos de desarrollo.

### 2.4 Build Script — spatial-index

**Decisión**: Remover `spatial-index` del array `modules` en `scripts/build-wasm.mjs`. Agregar comentario explicando que deberá añadirse cuando el crate tenga artefactos WASM completos y el frontend lo importe.

**Justificación**: El crate `rust-wasm/spatial-index` existe en el workspace Rust pero no tiene artefactos `public/wasm/spatial-index/spatial_index.js` comprometidos. El script fallaba en entornos sin wasm-pack instalado, rompiendo el build de contribuidores. La solución correcta es no incluirlo hasta que esté listo para producción.

### 2.5 CI Workflow — build-wasm.yml

**Decisión**: Documentar en `.github/workflows/build-wasm.yml` que `spatial-index` no se despliega todavía.

**Justificación**: El workflow solo construye y commitea artefactos de `route-calculator`. La documentación inline previene confusión futura sobre por qué `spatial-index` no está en el step de commit.

## 3. 📋 Justificación General

- **Calidad**: Los 8 comentarios de revisión de Copilot eran defectos reales que podrían causar fallos intermitentes en CI.
- **Mantenibilidad**: Documentación alineada con el código real reduce deuda técnica de comprensión.
- **Seguridad del pipeline**: Un build script que falla en ciertos entornos bloquea contribuciones.
- **Limpieza del repositorio**: Scripts temporales comprometidos dificultan la navegación del código.

---

## 4. 📊 Alternativas consideradas

| Alternativa | Razón de descarte |
|-------------|-------------------|
| Mantener `patch_rutas.cjs` con documentación | Los scripts ya cumplieron su función; mantenerlos agrega ruido |
| Hacer tolerante el check de artefactos en build-wasm.mjs | Más código de mantenimiento; mejor simplificar el array `modules` |
| Dejar las mutaciones de `globalThis` con comentarios | No resuelve el problema de aislamiento real |
